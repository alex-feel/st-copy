# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`st-copy` is a Streamlit custom component that renders a copy-to-clipboard button. It is published to PyPI as `st-copy`. Targets Python >=3.10 and Streamlit >=1.56.

## Architecture

This is a **two-half Streamlit Component** built on the **Custom Components v2** API (frameless — the frontend renders directly in the app DOM, not in an iframe).

- **Python side** (`src/st_copy/__init__.py`): a thin wrapper around `streamlit.components.v2.component`. `copy_button(...)` is the only public API — it forwards its arguments as the component `data` payload, auto-generates a UUIDv4 `key` if none is provided, and mounts with `width='content'` so the button aligns inline inside `st.container(horizontal=True)` (this fixes issue #27, which the old V1 iframe could not). The mounting command is created **lazily and cached** so `import st_copy` is safe outside a running Streamlit app. The fully-qualified component name `st-copy.st_copy` is resolved against the runtime metadata in `src/st_copy/pyproject.toml`.
- **Component manifest** (`src/st_copy/pyproject.toml`): a small file shipped *inside* the package that declares `[[tool.streamlit.component.components]]` with `name = "st_copy"` and `asset_dir = "frontend/dist"`. Streamlit's manifest scanner reads it at runtime to locate the built bundle and resolve the `js`/`css` globs. It is **not** the build config (that is the repo-root `pyproject.toml`) and carries no version.
- **Frontend side** (`src/st_copy/frontend/`): a tiny, **framework-free** TypeScript bundle (no React/Preact, no iframe). `src/index.ts` default-exports a `FrontendRenderer` from `@streamlit/component-v2-lib` that fills the Python-provided HTML skeleton (`.st-copy-btn`/`.st-copy-icon`/`.st-copy-label`), wires the click handler, and reports the result via `setTriggerValue('copied', ...)`. Theme awareness uses Streamlit's CSS custom properties (e.g. `--st-text-color`). Built by **Vite 8 (lib mode)** into `src/st_copy/frontend/dist/` as a single hashed `index-*.js` plus `index-*.css`; that `dist/` is what ships inside the wheel.
- **Local development**: v2 has **no dev-server URL mode**. Install the package editable (`uv pip install -e .`) and run `npm --prefix src/st_copy/frontend run dev` (`vite build --watch`) so Streamlit re-serves the rebuilt bundle; run the demo with `streamlit run examples/app.py`.
- **Packaging** (`pyproject.toml`): `hatchling` force-includes `src/st_copy/frontend/dist/**` (the rest of `frontend/` — sources and `node_modules` — is excluded from the wheel) plus the inner `pyproject.toml`. **The wheel is broken if the frontend bundle is missing/stale** — `tests/test_build.py` guards against this by greping the bundle for a known string.

## Common commands

All commands assume the repo root as cwd.

### Lint, type-check, format
```bash
uv run pre-commit run --all-files --show-diff-on-failure
```
Runs Ruff (`--fix`), `ty check` (Astral's type checker, invoked via `uv run`), ESLint on the frontend, JSON/whitespace hooks, `uv-lock`, and commitizen.

### Python tests
```bash
uv run pytest -vv --junitxml=report.xml --cov=src --cov-report=xml --cov-report=term

# Single test
uv run pytest tests/test_api.py::test_key_autogenerated_uuid4 -vv
```
`tests/test_api.py` replaces the cached v2 mounting command with a recorder (reloading the module to reset the cache) — there's no real Streamlit runtime needed. `tests/test_integration.py` boots the component via `streamlit.testing.v1.AppTest` (this requires the package to be installed so the component manifest is discoverable). `tests/test_build.py` validates the prebuilt JS bundle is present and current.

### Frontend tests / lint
```bash
npm --prefix src/st_copy/frontend test     # vitest (jsdom)
npm --prefix src/st_copy/frontend run lint # ESLint
```

### Build the frontend bundle (required before `uv build`)
```bash
npm ci    --prefix src/st_copy/frontend --no-audit --no-fund
npm run   --prefix src/st_copy/frontend build
```

### Build the wheel and verify the bundle is included
```bash
uv build
unzip -l dist/*.whl | grep -E 'frontend/dist/index-.*\.js'
```

### Run the demo app against a live frontend
```bash
# One-time: install the package editable so the component manifest is discoverable
uv pip install -e .

# Terminal 1 — rebuild the bundle into dist/ on every change (no dev server in v2)
npm --prefix src/st_copy/frontend run dev

# Terminal 2 — Streamlit demo (re-serves the rebuilt bundle on rerun)
streamlit run examples/app.py
```

## Conventions and constraints

- **Never bump versions manually.** `pyproject.toml`, `src/st_copy/frontend/package.json`, `src/st_copy/frontend/package-lock.json`, and `.release-please-manifest.json` are managed by the release-please GitHub workflow. Don't touch them in PRs. (The inner `src/st_copy/pyproject.toml` deliberately has no version.)
- **Conventional Commits 1.0** with the trimmed type set in `CONTRIBUTING.md` (`feat`, `fix`, `docs`, `test`, `ci`, `chore`). No scopes. `commitizen` enforces this in pre-commit.
- **Ruff config** (`pyproject.toml`): `target-version = py310`, `line-length = 127`, single quotes (inline + multiline), double quotes for docstrings, isort with `force-single-line = true` and `known-first-party = ["st_copy"]`.
- **Keep the bundle small.** The frontend is intentionally framework-free (no React/Preact) — adding a new JS runtime dependency is a real cost since it ships in the wheel. Prefer DOM/standard APIs.
- **`copy_button` is the public surface.** Any change to its signature requires updating `README.md` (the parameter table is duplicated there) and adding/updating tests in `tests/test_api.py`.
